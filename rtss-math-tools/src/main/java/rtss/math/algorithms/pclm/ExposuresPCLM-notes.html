PCLM Exposure-Weighted Mortality Rate Ungrouping

This note summarizes the key implementation point from the discussion: if one-year population or exposure by age is available, PCLM can and should use it when ungrouping grouped mortality rates.

1. Can PCLM use one-year population exposures?

Yes. The Rizzi–Gampe–Eilers PCLM framework includes an extension from grouped counts to grouped rates. In the basic histogram/counts version, the latent unknowns are one-year counts, and a grouped observation is just the sum of those latent counts within the bin.

For mortality rates, the latent unknowns are instead one-year rates:

m[x] = unknown one-year mortality rate at age x
E[x] = known one-year exposure / population at age x

The expected number of deaths at age x is:

expected_deaths[x] = E[x] * m[x]

Therefore, for a grouped age interval i, the model should aggregate expected deaths as:

mu[i] = sum over x in bin i of E[x] * m[x]

If the source table gives a grouped mortality rate, for example an ADH value in deaths per 1000 persons, convert it to an estimated grouped death count before fitting:

D[i] = grouped_rate_per_1000[i] / 1000 * sum(E[x] over x in bin i)

The PCLM then estimates one-year rates m[x] such that:

D[i] ≈ sum over x in bin i of E[x] * m[x]
Important: this makes the grouped rate an exposure-weighted average, not a simple arithmetic average of one-year rates.

2. Why this matters especially for ages 85+

For a wide old-age group such as 85+ or 85–100, population exposure falls sharply with age. Ages 85–89 usually carry much more weight than ages 95–100. Therefore a true mortality curve can rise substantially at the oldest ages while the grouped 85+ rate remains moderate.

The correct grouped-rate identity is:

m_group = sum(E[x] * m[x]) / sum(E[x])

not:

m_group = average(m[x])

If the algorithm treats the grouped mortality rate as a simple average over ages 85–100, it imposes the wrong constraint. That can force the old-age curve to be too low or produce artificial bends near the right boundary.

3. Does the R code in the Rizzi article include exposure?

The R code printed in Appendix 2 of the article does not include an explicit exposure argument. It demonstrates the basic PCLM for grouped counts / histogram ungrouping:

pclm <- function(y, C, X, lambda = 1, deg = 2, show = F)

That demo uses:

  • y: observed grouped counts;
  • C: composition matrix;
  • X: design matrix, often identity;
  • lambda: smoothing parameter.

In the demo, C is a 0/1 matrix. For example, C[i, x] = 1 if age x belongs to bin i. This is appropriate for grouped counts.

However, the function is general enough to support rates if the exposure weights are inserted into C before calling pclm.

4. Composition matrix for counts vs. rates

Case Meaning of latent gamma Composition matrix entry Observed y
Grouped counts One-year counts C[i, x] = 1 if age x belongs to bin i Observed count in bin i
Grouped rates One-year rates C[i, x] = E[x] if age x belongs to bin i Estimated death count in bin i

5. Implementation recipe

For mortality rates, implement a separate PCLM mode, conceptually:

input:
  grouped_rate[i]        // e.g. deaths per 1000 persons
  age_bin[i]             // e.g. 0, 1-4, 5-9, ..., 80-84, 85+
  exposure[x]            // one-year population/exposure by age

convert grouped rates to grouped death counts:
  D[i] = grouped_rate[i] / 1000.0 * sum(exposure[x] for x in age_bin[i])

build composition matrix:
  C[i][x] = exposure[x] if x in age_bin[i]
  C[i][x] = 0 otherwise

fit PCLM:
  D ≈ C * m

output:
  m[x] = estimated one-year mortality rates

After estimating one-year m[x], convert to q[x] if needed. The aggregation relationship is linear for central death rates / hazards, but not for probabilities of dying q[x]. Therefore, the rate-PCLM should preferably operate in m[x] space rather than directly in q[x] space.

6. Practical conclusion

If your current implementation passes grouped mortality coefficients to PCLM as ordinary binned values, with a 0/1 composition matrix, it is not using the available population-age distribution. It is effectively preserving simple bin averages, not exposure-weighted mortality rates.

The more principled fix is not merely to add a fake final bin or adjust the 80–84 / 85+ values, although those may still be useful diagnostics or secondary corrections. The core fix is to implement the rate version of PCLM:

C[i][x] = exposure[x]
y[i]    = grouped_rate[i] * total_exposure_in_bin

This is the formulation that lets PCLM use one-year population exposures when decomposing grouped age-specific mortality rates.
