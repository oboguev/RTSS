Идентификаторы
==============

RENGLON		уникальный номер строки / случая
ANIO		год
ENTIDAD		субъект федерации / штат; для страны — República Mexicana
CVE_GEO		географический код INEGI
ENTIDAD_FEDERATIVA	название субъекта федерации; фактически дублирует ENTIDAD
FECHA		дата вида YYYY-01-01, техническое представление года

Основные демографические величины
=================================

CRE_NAT		естественный прирост = рождения − смерти
CRE_SOC		социальный прирост, то есть миграционный баланс
CRE_TOT		общий прирост = естественный + социальный
DEF		смерти, число умерших
EDAD_MED	медианный возраст
NAC		рождения
POB_MIT_ANIO	население на середину года
HOM_MIT_ANIO	мужчины на середину года
MUJ_MIT_ANIO	женщины на середину года
IND_ENV		индекс старения
EVH		ожидаемая продолжительность жизни при рождении, мужчины
EVM		ожидаемая продолжительность жизни при рождении, женщины
EV		ожидаемая продолжительность жизни при рождении, всего

Миграция
========

EMI_EST		межштатные эмигранты, выбывшие в другие штаты Мексики
INM_EST		межштатные иммигранты, прибывшие из других штатов Мексики
MIG_NET_EST	чистая межштатная миграция = INM_EST − EMI_EST
EMI_INT		международные эмигранты
INM_INT		международные иммигранты
MIG_NET_INT	чистая международная миграция = INM_INT − EMI_INT

Технические колонки fn_*
========================

В CSV есть дополнительные поля:

fn_CRE_SOC, fn_CRE_TOT, fn_EMI_EST, fn_EMI_INT, fn_INM_EST, fn_INM_INT, fn_MIG_NET_EST, fn_MIG_NET_INT, fn_T_CRE_SOC, fn_T_CRE_TOT, fn_T_EMI_EST, fn_T_INM_EST, fn_T_MIG_NET_EST, fn_T_MIG_NET_INT.

Их нет в PDF-словаре. По содержанию файла они выглядят как числовые копии соответствующих колонок без префикса fn_, где значения ND заменены пустой строкой, а числовые значения записаны как числа с десятичной точкой. Например:

Исходная колонка	fn_-колонка		Смысл
CRE_SOC			fn_CRE_SOC		числовая версия CRE_SOC
EMI_INT			fn_EMI_INT		числовая версия EMI_INT
T_CRE_TOT		fn_T_CRE_TOT		числовая версия T_CRE_TOT
T_MIG_NET_INT		fn_T_MIG_NET_INT	числовая версия T_MIG_NET_INT

То есть для анализа лучше использовать обычные колонки, а fn_* можно использовать как уже очищенные числовые версии там, где в исходных колонках встречается ND.
