# Список исполняемых программ

## Основные программы

**EvalCountryTaxon** – расчитать погодовую величину населения и его движения для различных составных таксонов.
- Если флаг **DoCountMilitaryDeaths** = true, то учитываются военные смерти, иначе военные смерти не учитываются.

Сортировка территорий (губерний и областей):
- **ByBirthRate** – по рождаемости в 1913 году<br><br>
- **ByDeathRate** – по смертности в 1913 году<br><br>
- **ByGrowthRate** – по естественному приросту в 1913 году<br><br>
- **ByIncreaseOfGrowthRate** – по изменению темпа естественного прироста между средней величиной за 1896, 1897, 1899, 1900 и средней за 1911, 1912, 1913

В отличие от составных таксонов, по-территоральных (губернских и областных) данных военные смерти не учитываются, иммиграция в по-территоральных данных учитывается частично.

**ShowAreaValues** – распечатывает для запрашиваемых губерний или областей:
- погодовые значения оценок численности населения по ЦСК и УГВИ;
- оценку рождаемости и смертности исходя из данных УГВИ;
- оценку населения прогрессивно отсчитываемую от переписи 1897 года прибавлением естественного движения и баланса переселенческой миграции (но не других видов миграции), и величины рождаемости и смертности на базе этой оценки. 

**ShowAllAreaValues** – распечатывает названные значения для всех губерний и областей. 

**ShowAllAreaValuesRaw** – распечатывает значения из оцифрованных файлов сведений УГВИ без приложения к ним правок. Исправлены (уже в самих оцифрованных файлах) только заведомые опечатки, выявленные по перекрёстным суммам.

**ShowAllAreasValuesRaw** – распечатать погодовые значения для территорий, без слияния губерний и областей образованных после переписи 1897 года с губерниями, из состава которых они были выделены.

**CSK_VS_UGVI** –  сопоставить численность и движение населения по оценке ЦСК и по оценке УГВИ.

**EvalDivergence_CSK_50** – вычислить погодовое (для 1904-1914) расхождение между публиковавшимися ЦСК величинами естественного прироста по 50 губерниям и годового изменения численности их населения 
согласно соответственно "Движению Европейской части России" и "Ежегодникам России" ЦСК

**EvalDivergence_UGVI** – вычислить погодовое (для 1893-1913) расхождение между публиковавшимися УГВИ величинами естественного прироста и годового изменения численности населения по Империи: сравнение разницы между численностью населения в последовательные годы Y и Y+1 и (числа рождений минус числа смертей) за год Y.

**PrintTotalBoth** – печатает численность населения территорий по переписи 1897 года.

**RSFSR_1991_MigrationBalance** - печатает погодовой (для 1896-1916) миграционный баланс (сальдо) для территории РСФСР-1991 и для суммы областей не входящих в РСФСР-1991; контролирует урвановешенность балансов и показывает погодовую величину переселенческого потока извне границ РСФСР-1991 в эти границы.

## Контроль качества данных

**FlagOutliers** – перечислить сильные уклонения в числе рождений или смертей в губернии от соседних лет.

**FlagUnderRegistration** – выявить территории с заниженной регистрацией рождений и смертей в начале периода сравнительно с концом периода; некоторые известные проблемные территории, перечисленные в программном файле, пропускаются, это: Бакинская с Баку, Дагестанская обл., Елисаветпольская, Закаспийская обл., Карсская обл., Кутаисская с Батумской, Семиреченская обл., Сыр-Дарьинская обл., Ферганская обл., Черноморская, Самаркандская обл.

**ShowMissingBD** – перечислить, для каких территорий и в какие годы нет данных о смертях или рождениях.

## Не предназначены для обычного использования

**LoadData** – используется при отладке загрузки данных.

**Show1898** – контролирует сравнительную величину показателей (таких как число рождений) в 1898 году относительно двух соседних лет, по территориям.

**ShowData** – загрузить и распечатать выбранные виды данных по выбранным территориям и годам с опциями прилагаемой коррекции; возможные опции перечислены в LoadData.

## Опции загрузки данных

Опции LoadData употребляются в программах EvalCountryTaxon, ShowXXX и других и означают:

- **ADJUST_FEMALE_BIRTHS** – внести поправку на недорегистрацию рождений девочкек: исправить заниженное число рождений для женщин сделав его >= числу мужских рождений / 1.06.<br><br>
- **FILL_MISSING_BD** – заполнить пробелы в сведениях о числе рождений и смертей; приложить коррекции из файла ugvi/PatchYearData.xslx; скорректировать на недоучёт регистрации рождений и смертей иудеев.<br><br>
- **MERGE_CITIES** – удалить отдельные записи для городов (Москвы, Петербурга, Варшавы, Одессы, Николаева, Севастополя, Баку, Ростовского-на-Дону градоначальства), влив их население в состав соответствующих губерний.<br><br>
- **MERGE_POST1897_REGIONS** – слить губернии и области образованные после переписи 1897 года с губерниями, из состава которых они были выделены, образовав новые составные записи для сохранения непрерывности истории (Иркутская + Камчатская -> Иркутская с Камчатской; Кутаисская  + Батумская -> Кутаисская с Батумской; Люблинская + Седлецкая + Холмская -> Люблинская с Седлецкой и Холмской).<br><br>
- **EVAL_PROGRESSIVE** – вычислить прогрессивную оценку населения отсчётом от переписи 1897 года (только для УГВИ) и сохранить её в поле progressive_population, параллельно собственной оценке УГВИ.<br><br>
- **VERIFY** – проверка внутренней согласованности загруженных данных; используется при отладке.<br><br>


## Общий образ вычисления EvalCountryTaxon

- загружает данные, прмменяя все опции LoadData, кроме VERIFY;<br><br>

- вычисляет численность губерний и областей населения на начало 1896 года по прогрессивной оценке;<br><br>

- производит правку расчёта УГВИ для Дагестана, перебазируя его от данных переписи 1897 года, а не ошибочной величины использованной УГВИ;<br><br>

- использует для прогрессивного расчёта стаблизированные участки для Карсской области, Терской области Тифлисской губернии, Закаспийской, Семиреченской, Сыр-Дарьинской и Ферганской областей; пересчёт замещает не только progressive_population, но и population, однако последняя структура затем не используется;<br><br>

- проводит исправление для Бакинской губернии с Баку:<br><br>
     - предварительно устраняет в оценке УГВИ осцилляции путём экспоненциального интерполирования (с постоянным годовым темпом роста) численности населения между значениями 1903 и 1914 годов;<br><br>
     - выполняет погодовое усреднение численности населения из двух оценок: прогрессивной и УГВИ, и принимает результат как новую прогрессивную оценку;<br><br>

- выполняет правку прогрессивного расчёта для Самаркандской области:<br><br>
     - использует для неё расчёт УГВИ (1896-1901), затем ЦСК (1904-1914);
     - численность в 1902 и 1903 г. интерполируется между 1901 и 1904;<br><br>

- выполняет правку прогрессивного расчёта для Уральской области: использует прогрессивный расчёт (1896-1903), затем расчёт ЦСК (1905-1914), и среднее для 1904;<br><br>

- для Приморской области исправляет численность населения по прогрессивному расчёту для начальных лет, повышая число рождений и смертей в 1896-1898 гг. до соответствия средним уровням рождаемости и смертности за 1899-1903 годы, для компенсации общего недоучёта в в 1896-1898 гг.<br><br>

- следующие территории исключаются из подсчёта рождаемости и смертности для сводного таксона (из-за непригодности их данных о рождаемости и смертности) и используются только для подсчёта совокупного населения таксона:<br><br>
     - Дагестанская обл.
     - Елисаветпольская
     - Бакинская с Баку
     - Кутаисская с Батумской
     - Акмолинская обл.
     - Тургайская обл.
     - Самаркандская обл.
     - Семипалатинская обл.
     - Уральская обл.
     - Приморская обл.<br><br>

- для таксона РСФСР-1991 добавляет коррекцию на миграционный приток в Черноморскую губернию, повышая численность губернии по прогрессивному расчёту на величину принятого в каждом году притока;<br><br>

- суммирует прогрессивные расчёты для всех территорий входящих в составной таксон (население, число рождений и число смертей в них для каждого года); для территорий входящих в таксон лишь частично, а не полностью, применяется долевой процентный множитель, на который территория входит в таксон;<br><br>

- дальнейшие действия выполняются с полученной суммой прогрессивных расчётов;<br><br>

- если DoCountMilitaryDeaths включён, то добавляет военные смерти за 1904 и 1905 годы пропорционально общим потерям Империи и соотношению численности населения таксона к населению Империи;<br><br>

- если DoCountMilitaryDeaths включён, то для таксона РСФСР-1991 добавляет к 1914 году число военных смертей; для других таксонов расчёт ограничивается по времени рубежом 1914 года;<br><br>

- добавляет часть иммиграции не разбиваемую по губерниям.<br><br>
